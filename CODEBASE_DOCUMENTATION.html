1. System Overview & Architecture

The Zomato platform is a full-stack food delivery application with two main user types: Customers and Restaurant Partners. The system handles user authentication, food catalog management, cart operations, and payment processing.

Protected Route Middleware

auth() Middleware:
1. Extract token from req.cookies.token
2. If no token → 401 "token missing"
3. Verify JWT: jwt.verify(token, SECRET_KEY)
4. If invalid → 500 "failed to authenticate"
5. Attach decoded payload: req.user = {id, role}
6. Call next() → route handler executed
isPartner() Middleware:
1. Check req.user.role === "partner"
2. If not partner → 403 "Access denied. Partners only."
3. If partner → next()
isOwner() Middleware:
1. Get partnerId from req.params
2. Find partner in database
3. Check partner._id === req.user.id
4. If owner → next()
5. If not → 403 "Not the owner"

3. Database Models & Relationships

Database Schema Relationships

erDiagram USER ||--o{ CART : has USER ||--o{ ORDER : places USER ||--o{ LIKE : gives PARTNER ||--o{ MENUE : offers PARTNER ||--o{ REEL : uploads MENUE ||--o{ REEL : "featured in" MENUE ||--o{ LIKE : receives MENUE ||--o{ ORDER_ITEM : "ordered in" ORDER ||--o{ ORDER_ITEM : contains ORDER ||--|| PAYMENT : linked_to CART ||--o{ CART_ITEM : contains CART_ITEM ||--|| MENUE : references REEL ||--o{ LIKE : "has many"

User Model

Collection: users

{ _id: ObjectId, fullName: String (required), email: String (required, unique), password: String (hashed), phoneNumber: String, isEmailVerified: Boolean (default: false), isPhoneNumberVerified: Boolean (default: false), resetToken: String, expire: Date, userType: String (default: "customer"), createdAt: Timestamp, updatedAt: Timestamp }

Partner Model

Collection: partners

{ _id: ObjectId, ownerName: String (required), email: String (required, unique), password: String (hashed), restaurantName: String (required), location: String (required), phoneNumber: String, isEmailVerified: Boolean (default: false), isPhoneNumberVerified: Boolean (default: false), resetToken: String, expire: Date, userType: String (default: "partner"), createdAt: Timestamp, updatedAt: Timestamp }

Menu Item Model (Menue)

Collection: menus

{ _id: ObjectId, name: String (required), description: String, price: Number (required), foodPartener: ObjectId (ref: Partner, required), quantity: Number, cuisine: Enum ["Indian", "Chinese", "Italian", "Mexican", "Continental", "Other"], image: String (image URL from storage), quantityUnit: Enum ["pcs", "cup", "ml", "gram", "kg", "plate"], currency: Enum ["INR", "USD", "EUR"] (default: "INR"), status: Enum ["live", "low stock", "out of stock"] (default: "live"), createdAt: Timestamp, updatedAt: Timestamp }

Food Reel Model

Collection: reels

{ _id: ObjectId, reelName: String (required), description: String, video: String (video URL from storage, required), likeCounts: Number (default: 0), likes: [ObjectId] (ref: User, array of liking users), foodPartener: ObjectId (ref: Partner, required), foodItemId: ObjectId (ref: Menue, required), createdAt: Timestamp, updatedAt: Timestamp }

Cart Model

Collection: carts

{ _id: ObjectId, userId: ObjectId (ref: User, required), items: [{ foodItemId: ObjectId (ref: Menue, required), quantity: Number (default: 1), partnerName: String (required) }], createdAt: Timestamp, updatedAt: Timestamp }

Order Model

Collection: orders

{ _id: ObjectId, orderedItems: [{ menuId: ObjectId (ref: Menue, required), quantity: Number (required) }], userId: ObjectId (ref: User), amount: { price: Number (required), currency: String (default: "INR") }, createdAt: Timestamp, updatedAt: Timestamp }

Payment Model

Collection: payments

{ _id: ObjectId, orderId: String (Razorpay Order ID, required), paymentId: String (Razorpay Payment ID), signature: String (Razorpay Signature), price: { amount: Number (required), currency: String (default: "INR") }, status: Enum ["pending", "completed", "failed"] (default: "pending"), createdAt: Timestamp, updatedAt: Timestamp }

4. Backend API Structure

Authentication Routes (/api/user)

Method Endpoint Controller Function Auth Description
POST /register registerUser() ❌ Register new customer account
POST /login login() ❌ Login user/partner, send OTP
GET /logout logoutUser() ✅ Clear auth cookie, logout user
GET /me getMe() ✅ Get current user/partner info
POST /send-otp sendOtp() ❌ Send OTP to phone number
POST /verify-otp verifyOtp() ❌ Verify OTP, set phone verified
POST /forgot-password forgetPassword() ❌ Generate & email reset link
POST /reset-password/:token resetPassword() ❌ Reset password with token
POST /foodpartener/register registerPartener() ❌ Register restaurant partner
GET /foodpartener/logout logoutPartener() ✅ Logout restaurant partner

Food & Menu Routes (/api/food)

Method Endpoint Controller Function Auth Description
GET /partners getPartners() ❌ Get all restaurant partners
GET /partner/:partnerId getPartnerById() ❌ Get specific partner details
GET /menue/:partnerId getMenueByPartnerId() ❌ Get menu items for partner
GET /fooditem/:itemId getItembyId() ❌ Get single food item details
POST /addmenueitem/:partnerId addMenuItems() ✅ Partner adds menu item
PATCH /menueitem/:partnerId/status/:itemId updateMenuItemStatus() ✅ Update item status
GET /reels getAllReels() ❌ Get all food reels
POST /addfooditemreel/:partnerId addFooodItemsReel() ✅ Partner uploads food reel
GET /reelcount/:partnerId getReelCount() ❌ Get reel count for partner
GET /likes?foodId=X getLikeCount() ❌ Get like count for food item
POST /likes likeCount() ❌ Toggle like on food item

Cart Routes (/api/cart)

Method Endpoint Controller Function Auth Description
POST /add addToCart() ✅ Add item to user's cart
GET / getCartItems() ✅ Fetch user's cart items
POST /remove removeFromCart() ✅ Remove item from cart
POST /decrease decreaseQuantity() ✅ Decrease item quantity
POST /clear clearCart() ✅ Clear entire cart

Payment Routes (/api/payment)

Method Endpoint Controller Function Auth Description
POST /checkout checkOut() ❌ Create order, initiate Razorpay payment
POST /verify verify() ❌ Verify Razorpay signature, confirm payment

8. Key Features Summary

✅ Stock Management: live / low stock / out of stock statuses

✅ Quantity Management: Increase, decrease, or remove items

Payments

✅ Order History: Users can view past orders

Partner Dashboard

✅ Menu Management: Add, update, delete food items

✅ Video Reels: Upload promotional food videos
